HL7 Message Types

Message Types are broad categories of event triggers or reasons why an HL7 message may go out an interface to a downstream system.

Examples of message types include ADT (Admission, Discharge and Transfer), ACK (General Acknowledgment Message), and DFT (Detailed Financial Transactions)

Hl7 Trigger Events

Trigger Events are specific HL7-defined or user-defined events that occur by a clinical or business use in which something occurs to the UI (user interface) which then triggers a message to be sent about the event details.

Examples include an ADT^A01 message, which signals to a downstream or neighboring application within a hospital network that a patient has been admitted.

Hl7 Message Segments

Each line within an HL7 message is made up of segments. The segments themselves are also semantic in that they categorize the type of information being sent.

The MSH segment, for example exists across all message types and is the Message Header with basic detail including who sent the message, time of message being sent, the message type, etc.
